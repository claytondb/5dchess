## Design Philosophy

**Core Principle: "Complexity Through Layers, Not Clutter"**

The fundamental UX challenge is managing 5 dimensions of information:
1. X/Y position (2D space)
2. Z depth (3D combat)
3. Time (temporal position)
4. Timeline (parallel universes)

**Solution: Progressive Disclosure + Focus System**

Players see ONE timeline at full detail. Others exist as contextual ghosts until needed. The UI reveals complexity only when the player asks for it.

### 6.1 Mouse + Keyboard (Primary)

**Movement & Combat:**
| Input | Action |
|-------|--------|
| `W A S D` | Ship movement (relative to camera) |
| `Mouse` | Aim weapons / Look around |

| `Right Click` | Fire secondary weapon |
| `Mouse Wheel` | Zoom in/out |
| `Middle Click` | Lock onto target |
| `Shift` | Boost / Afterburner |
| `Ctrl` | Slow down / Precision mode |

**Time Travel:**
| Input | Action |
|-------|--------|
| `SPACE` (hold) | Open Temporal Wheel (time slows) |
| `SPACE` + Drag | Select time/timeline destination |
| `SPACE` release | Confirm jump |
| `Q` | Quick-rewind (5 seconds) |
| `E` | Quick-forward (5 seconds) |
| `Tab` | Cycle active timeline |
| `~` | Return to origin timeline |
| `T` | Open full Timeline Navigator |

**Weapons & Abilities:**
| Input | Action |
|-------|--------|
| `1 2 3 4` | Select weapon |
| `R` | Reload / Recharge |
| `F` | Use special ability |
| `G` | Deploy temporal beacon |
| `X` | Create timeline branch |

**UI & System:**
| Input | Action |
|-------|--------|
| `ESC` | Pause menu |
| `M` | Toggle timeline minimap |
| `I` | Inventory / Loadout |
| `J` | Mission objectives |
| `F1` | Help / Controls |
| `F5` | Quick save |
| `F9` | Quick load |

### 7.3 Complexity Gating

**Progressive Unlock System:**

| Player Level | Unlocked Features |
|--------------|-------------------|
| 1-3 | Quick rewind only, 1 timeline |
| 4-6 | Full time travel, 2 timelines max |
| 7-10 | Branching, 3 timelines max |
| 11-15 | Full mechanics, 4 timelines |
| 16+ | Expert mode: 5+ timelines, paradox hard mode |

**Difficulty Tiers:**

```
CASUAL MODE:
- Generous time slow during Temporal Wheel
- Auto-manage secondary timelines
- No paradox failures (warnings only)
- Simplified timeline view

STANDARD MODE:
- Normal time slow
- Must check timelines manually
- Paradoxes cause penalties
- Full timeline view

HARDCORE MODE:
- Minimal time slow
- Timelines can collapse = mission fail
- Strict paradox rules
- Information-dense UI
```
